Прочитай SPEC.md в корне репозитория целиком. Это спецификация скилла openspec-task-explain, дополняющего пайплайн OpenSpec Skills.

По этой спецификации сгенерируй ровно два файла и ничего больше:

1. skills/openspec-task-explain/SKILL.md — основной файл скилла, в формате один-в-один как у openspec-design/SKILL.md из ai-risk-hub-inner-skill-hub. Структура и точное содержание секций — в SPEC.md, разделы 4.1-4.9.

2. skills/openspec-task-explain/templates/briefing.md — шаблон выходного briefing-документа. Точная структура — в SPEC.md, раздел 5.

Важно:
- Никаких других файлов не создавай. Не пиши README скилла, не создавай тесты, не трогай examples/.
- Содержимое секций SKILL.md воспроизводи в точности как указано в SPEC.md разделах 4.1-4.9. Это не "вдохновение", это требование.
- Ни одной цитаты в SKILL.md не должно появиться "из головы" — все формулировки берутся из SPEC.md.
- Перед финализацией прогони себя по списку антипаттернов из раздела 11 SPEC.md и проверь что ни один из них не воспроизведён.

Когда сгенерируешь — выведи в чат список созданных файлов и краткую сводку: какие секции SKILL.md заполнил, какие плейсхолдеры использовал в briefing.md.
